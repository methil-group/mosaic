# Mosaic Tool Protocol 🧩

This document defines the standardized XML-wrapped JSON protocol used for model-to-tool communication in Mosaic.

## 🛠 Tool Calling Format

When the model wants to call a tool, it MUST use the `<tool_call>` tag containing a JSON object.

### Schema
```json
{
  "name": "string",
  "arguments": "object"
}
```

### Example
```xml
<tool_call>
{
  "name": "read_file",
  "arguments": {
    "path": "src/main.py"
  }
}
</tool_call>
```

---

## 📥 Tool Response Format

The system provides the tool result in a `<tool_response>` tag containing a JSON object.

### Schema
```json
{
  "tool_call_id": "string",
  "name": "string",
  "content": "object"
}
```

- **tool_call_id**: A unique identifier generated by the system for tracking.
- **name**: The name of the tool that was called.
- **content**: A JSON object containing the tool output. If the tool returns plain text, it will be wrapped in a `{"message": "..."}` or `{"output": "..."}` object.

### Example
```xml
<tool_response>
{
  "tool_call_id": "chatcmpl-tool-86225be549df082c",
  "name": "write_file",
  "content": {
    "bytes_written": 7660,
    "dirs_created": false
  }
}
</tool_response>
```

---

## 🔄 Interaction Loop

1. **Model** outputs a `<tool_call>`.
2. **System** masks the tool call from the UI (optional), executes the tool, and generates an ID.
3. **System** appends a `<tool_response>` to the conversation history.
4. **Model** receives the response and continues reasoning.
